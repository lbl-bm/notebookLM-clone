## 🔧 技术实现细节

### Dense 检索 (向量相似度)

**技术栈**:
- PostgreSQL pgvector 扩展
- Cosine 相似度度量
- 向量维度: 1024

**性能**:
- 延迟: <100ms
- 相似度范围: [0-1]
- 支持自定义阈值

**实现**:
```typescript
// 调用 PostgreSQL RPC 函数
SELECT * FROM match_document_chunks(
  notebook_id,
  query_embedding,
  topK,
  threshold
)
```

### Sparse 检索 (全文搜索)

**技术栈**:
- PostgreSQL tsvector
- ts_rank 相关度评分
- GIN 索引加速

**性能**:
- 延迟: <100ms
- 相关性范围: 归一化到 [0-1]
- 使用固定上界 (1.0) 归一化

**实现**:
```typescript
// 使用 plainto_tsquery 避免特殊字符问题
SELECT * FROM document_chunks
WHERE content_tsv @@ plainto_tsquery('english', query_text)
ORDER BY ts_rank(content_tsv, plainto_tsquery(...)) DESC
```

### RRF 融合 (Reciprocal Rank Fusion)

**算法**:
```
score = Σ (1 / (k + rank_i))
其中 k = 60 (标准参数)
```

**优点**:
- 无需调整权重即可融合
- 对排名相对位置敏感，不对绝对分数敏感
- 对离群值不敏感

**实现**:
```typescript
// 1. 为两路结果计算 RRF 分数
// 2. 累加既在 Dense 又在 Sparse 的结果
// 3. 去重并按融合分数排序
```

### 灰度控制系统

**组件**:
- FeatureFlagManager: 单例，管理灰度开关
- CanaryMetricsLogger: 收集性能和质量指标
- 一致性哈希: 确保同一用户多次调用结果一致

**功能**:
- 按百分比灰度 (1% → 10% → 50% → 100%)
- 白名单/黑名单支持
- 5 分钟缓存 TTL
- 自动指标收集
