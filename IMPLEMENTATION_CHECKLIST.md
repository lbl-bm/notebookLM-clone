### P1.1 ✅ RAG 多路召回与精排 (3-5d)

**目标**: 提升检索准确率 15-25%, 首字延迟下降 30%

  - 需求: 支持 Dense + Sparse + Query Expansion 三路召回

  - 关键依赖: -

- [ ] **任务 P1.1.2** - 实现 BM25 混合检索 (1d)
  - 位置: `lib/rag/bm25-retriever.ts` (新建)
  - 需求: 基于 pgvector `ts_vector` 全文搜索
  - 验收标准: 可检索 BM25 top10 + 相似度分数
  - 关键依赖: P1.1.1

- [ ] **任务 P1.1.3** - 实现 RRF 融合策略 (1d)
  - 位置: `lib/rag/fusion.ts`
  - 需求: Reciprocal Rank Fusion 加权融合
  - 验收标准: 融合后排序准确率 +15%
  - 关键依赖: P1.1.2

- [ ] **任务 P1.1.4** - 集成跨越编码器精排 (1-1.5d)
  - 位置: `lib/rag/ranking.ts` (新建)
  - 需求: 可选的二阶段精排 (快速版仅用相似度排序)
  - 验收标准: P@5 提升 10%+
  - 关键依赖: P1.1.3

- [ ] **任务 P1.1.5** - Chat API 集成 + 灰度开关 (0.5-1d)
  - 位置: `app/api/chat/route.ts`
  - 需求: 集成多路召回, 添加 `useMultipleRetrieval` 开关

### P1.2 ✅ 智能置信度与拒答机制 (2-3d)

**目标**: 幻觉率降低 40-50%, 用户信任度上升

#### 任务分解
- [ ] **任务 P1.2.1** - 设计置信度分项评分体系 (0.5d)
  - 位置: `lib/rag/confidence.ts` (新建)
  - 需求:
    - relevance_score (向量相似度)
    - coverage_score (证据覆盖度)
    - consistency_score (一致性)
    - source_diversity_score (来源多样性)
  - 验收标准: 设计文档 + 评分公式推导
  - 关键依赖: -

- [ ] **任务 P1.2.2** - 实现置信度计算器 (1d)
  - 位置: `lib/rag/confidence.ts`
  - 需求: 集成上述4项评分, 输出 combined_confidence
  - 验收标准: 置信度与实际准确率相关系数 >0.7
  - 关键依赖: P1.2.1

- [ ] **任务 P1.2.3** - 实现拒答判定逻辑 (0.5d)
  - 位置: `lib/rag/confidence.ts`
  - 需求:
    - confident (≥0.8): 直接回答
    - uncertain (0.6-0.8): 保守回答
    - no_evidence (<0.6): 拒答
  - 验收标准: 拒答准确率 >80%
  - 关键依赖: P1.2.2

- [ ] **任务 P1.2.4** - Chat API 集成 + UI 展示 (0.5-1d)
  - 位置: `app/api/chat/route.ts` + `components/ChatMessage.tsx`
  - 需求: 返回 confidenceScore 字段, 前端展示指示器
  - 验收标准: UI 可见置信度 + 拒答提示
  - 关键依赖: P1.2.3
